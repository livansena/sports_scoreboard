# ⚽ Sports Scoreboard

A web application built with Ruby on Rails that manages football teams, match results, and automatically generates league standings based on real competition rules.

The project was developed to strengthen my backend development skills using Ruby on Rails, PostgreSQL, and MVC architecture while implementing business logic commonly found in real-world sports management systems.

---

## 🚀 Project Overview

Sports Scoreboard allows users to:

* Manage football teams
* Register match results
* Track team performance
* Automatically calculate standings
* View recent matches and rankings through a centralized dashboard

The application demonstrates how business rules can be translated into backend logic, creating a dynamic league table similar to real football competitions.

---

## ✨ Features

### Team Management

Complete CRUD functionality for football teams.

Users can:

* Create teams
* View team lists
* Edit team information
* Remove teams

Each team is stored in PostgreSQL and displayed through a clean and responsive interface.

---

### Match Management

Users can register football matches by selecting:

* Home team
* Away team
* Home score
* Away score

All match data is persisted in the database and displayed in the matches section.

---

### Automatic Standings Calculation

One of the core features of the application.

Whenever a match is registered, the system automatically updates:

* Points
* Wins
* Draws
* Losses
* Goals Scored
* Goals Conceded
* Goal Difference

This business logic allows league standings to be generated dynamically without manual updates.

---

### Dashboard

A centralized dashboard provides:

* Recent matches
* Current league standings
* Team performance overview

The interface follows a modern sports-inspired design with a dark theme.

## 🏗 Architecture

The project follows the Ruby on Rails MVC architecture:

* Models handle business rules and data persistence
* Controllers manage application flow
* Views present information to users
* Active Record manages database relationships

Key concepts practiced:

* MVC Architecture
* CRUD Operations
* Active Record Associations
* Database Design
* Business Logic Implementation
* RESTful Routing

---

## 📊 Database Relationships

Main entities:

### Team

Stores football team information.

### Match

Stores:

* Home team
* Away team
* Goals scored
* Match result

Relationships are managed using Active Record associations.
